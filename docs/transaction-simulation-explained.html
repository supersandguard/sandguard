Transaction Simulation Explained

February 2, 2026 Technical Guide Simulation

You're about to sign a transaction that will swap 1,000 USDC for ETH through a complex DeFi protocol. The interface shows you'll receive "â‰ˆ 0.42 ETH," but how do you know that's accurate? What if there's slippage? What if the transaction fails and you still pay gas?

Transaction simulation answers these questions by running your transaction in a safe, isolated environment before you commit to it on the real blockchain. Think of it as a "practice run" that shows you exactly what will happen without any risk.

How Transaction Simulation Works

At its core, transaction simulation uses blockchain forking technology to create a perfect copy of the current blockchain state, then runs your transaction against that copy to see what happens.

Step 1: Fork the Blockchain

The simulation service creates an exact copy of the blockchain at the current block height. This fork contains all the same smart contracts, account balances, and state that exists on the real chain at that moment.

Step 2: Execute Your Transaction

Your transaction is executed against this forked environment. The virtual machine runs all the same code paths, makes all the same calls, and modifies state exactly as it would on the real blockchain.

Step 3: Capture the Results

The simulation engine records everything that happens: balance changes, gas usage, events emitted, internal transactions, and whether the transaction succeeds or reverts.

Step 4: Present the Outcome

All this data is formatted into human-readable results showing exactly what your transaction will do before you sign it.

The Technology Stack

Node Infrastructure: Tenderly runs dedicated blockchain nodes for all major networks. These nodes maintain complete blockchain state and can create forks instantly.

Virtual Machine: Each simulation runs in an isolated EVM (Ethereum Virtual Machine) instance. This ensures your simulation can't affect other users or the real blockchain.

State Management: The fork includes not just balances, but all smart contract state: AMM pool reserves, lending protocol positions, governance votes, NFT ownership, etc.

Gas Simulation: The simulation tracks exact gas usage for every operation, giving you accurate gas estimates including the impact of network congestion.

âš ï¸ Simulation Limitations

Simulations show what would happen if the transaction executed right now. But blockchain state changes constantly, so:

  • Large trades may have different slippage when actually executed
  • Gas prices fluctuate between simulation and execution
  • MEV bots might front-run your transaction
  • Network congestion can cause failures not shown in simulation

Always use recent simulations and understand they're estimates, not guarantees.

Dynamic vs. Static Costs

Smart contract operations have two types of gas costs:

Static costs are predictable â€” things like basic arithmetic, memory allocation, and simple function calls. These can be estimated without simulation.

Dynamic costs depend on blockchain state at execution time:

  • Storage operations: Writing to an empty storage slot costs 20,000 gas; updating an existing slot costs 5,000 gas
  • External calls: Gas costs vary based on the target contract's complexity
  • Loop iterations: The number of iterations depends on current state (e.g., how many positions need to be liquidated)
  • Conditional logic: Different code paths have different gas costs

Simulation captures all these dynamic costs by actually running the code.

Why You Should Simulate Every Transaction

Catch Errors Before They're Expensive

Transaction reverts on mainnet still cost gas. A failed transaction might cost $50+ in gas fees with nothing to show for it. Simulation catches these failures for free.

Verify UI Accuracy

Dapp interfaces sometimes show incorrect information due to bugs, outdated data, or intentional deception. Simulation gives you ground truth about what will actually happen.

Understand Total Costs

Many protocols have hidden fees not disclosed in the UI. Simulation shows exactly how much you'll pay in gas, protocol fees, slippage, and MEV.

âœ… Real Example: Avoiding a Costly Mistake

A user was about to provide liquidity to a "new DeFi protocol" promising 400% APY. The UI showed they would receive LP tokens worth $10,000 for their $10,000 deposit.

Simulation revealed the truth: the transaction would succeed, but the LP tokens received would be worth exactly $0. The protocol's "mint" function contained malicious code that granted worthless tokens.

Simulation cost: $0. Avoided loss: $10,000.

Plan Complex Strategies

DeFi strategies often involve multiple steps across different protocols. Simulation lets you test the entire strategy before committing funds:

  • Will the leverage position be profitable after fees?
  • What's the liquidation price after all transactions?
  • How much gas will the complete strategy cost?
  • What if one of the intermediate transactions fails?

Common Simulation Patterns

Token Swaps

What to check: Actual output amount, slippage percentage, fee breakdown, and gas cost relative to trade size.

Liquidity Operations

What to check: LP token value, impermanent loss risk, position health after deposit/withdrawal.

Lending & Borrowing

What to check: Collateral ratios, liquidation thresholds, interest rate changes, position sustainability.

Governance Actions

What to check: Vote weight calculation, proposal execution effects, token lock periods.

NFT Transactions

What to check: Ownership transfer, royalty payments, marketplace fees, authenticity verification.

Simulation Best Practices

Always Use Fresh Simulations

Blockchain state changes rapidly. A simulation from 10 minutes ago may no longer be accurate, especially for DEX trades or lending protocols.

Understand Your Risk Tolerance

Decide what level of slippage, gas cost, and outcome variance you're comfortable with before simulating. This helps you interpret results objectively.

Compare Multiple Scenarios

Test different gas prices, slippage tolerances, and route options. The "optimal" choice depends on your priorities (speed vs. cost vs. success probability).

Read the Full Report

Don't just look at the final balance changes. Review gas costs, intermediate steps, and any warnings or unusual patterns.
